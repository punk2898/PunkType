# PunkType 开放架构 / Roadmap

> 本文固化「开放方向」的设计思路与路线。核心判断：作为开源项目，PunkType 的根本优势不是再堆一个功能，而是**让别人能往里加东西**。我们已有的「全可自定义」（档位顺序、提示词、模型、词典、App 规则）本质上**全是配置数据**，把它做成可分享、可扩展，就是闭源竞品（Typeless / superwhisper 等）结构上学不来的护城河。

---

## 一、三层开放架构（总纲）

后续所有功能都往这张图里挂，不做零散功能，而是搭一个开放骨架：

```
① 输入层    识别引擎（本机 / Apple 快速 / Whisper）        —— 已可配
② 处理层    提示词 / 模型 / 档位（= 模式包）               —— 已可配；「配置包」让它可分享
③ 后处理层  出字之后的额外动作（Hook）                     —— 新增
             └─ 官方 Hook #1：记事本（自动分类归档）
             └─ 以后：翻译并存档 / 推送到 Obsidian / 调用快捷指令 / …
```

- **配置包**装的是 ②（并可声明启用哪些 ③）。
- **记事本**是 ③ 的第一个官方实现。
- 任何「出字后再做点啥」都是 ③ 上的一个新 Hook：官方先做，成熟后再开放第三方。

---

## 二、模式包（配置包，②层的分享）

### 是什么

一个「模式 / 包」= 一坨**纯数据，没有代码**（因此零安全风险、零运行时）。例如不同的提示词人格：正经、深思熟虑、调情、诡辩……差异全在提示词。

### 包的结构（草案）

```jsonc
{
  "id": "sophist-mode",
  "name": "诡辩模式",
  "icon": "🗣️",
  "version": "1.0",
  "author": "punk2898",
  "tier": {
    "prompt": "……核心提示词……",
    "model": "deepseek-v4-pro",
    "sttEngine": "apple-fast",
    "streaming": true
  },
  "appRules": [ /* 可选：哪些 App 自动启用 */ ],
  "dictionary": [ /* 可选：专属词条 */ ],
  "hooks": [ /* 可选：启用哪些后处理 */ ]
}
```

### 导出 / 导入

- 导出：把当前一档（或全部设置）打成一个 `.json`。
- 导入：读进来变成一个可选档位 / 一套配置。
- 这正是 [Espanso 包](https://espanso.org/docs/packages/basics/) 的玩法——可分享、可积累、可审计。

---

## 三、分发策略

原则：**官方包优先，第三方靠后；商城太早，先用 GitHub。**

- **官方优先**：早期由官方精选几个高质量模式包，质量与安全可控，等生态起来再开放第三方贡献。
- **GitHub 即轻量商城**（零后端）：仓库里放
  - `packs/` 目录存放各模式包 `.json`
  - 一个 `packs/index.json` 清单（列出官方包：名称、描述、文件名、版本）
  - App 内：拉取清单 → 列表展示 → 一键导入
- 等真的有规模了，再升级成网页商城不迟。Espanso 自己就是 GitHub Releases 起家。

---

## 四、后处理 Hook 层（③）

「出字完成后，能不能再多做一步处理？」——这就是 Hook。

- 定义一个**输出后的管道阶段**：拿到最终文本 → 依次跑启用的 Hook。
- Hook 不阻塞出字（异步），跟现有的「异步抽词 / 风格学习」同一套路。
- 官方先做内置 Hook；远期可开放「调用用户自己的脚本 / macOS 快捷指令」给极客（opt-in，安全边界清晰）。

## 五、记事本（官方 Hook #1）

### 概念

把你**日常口述的内容**，顺手收进一个**私密、本地**的记事本：自动分类打标签、可编辑、可搜索、可导出。它是日常打字的**副产品**——你本来就在到处语音输入，零额外动作就攒出一份个人记录。

主要面向**工作场景**：你每天说出去的内容本身就是工作轨迹，被动攒下来后再每日提炼，等于有人在旁边帮你记工作日志。

### 两层结构

1. **流水（原始条目）**：被动收录**所有**口述，每一条都保留（时间、来源 App、文本）。这是真相底座，不丢。
2. **每日总结（自动生成）**：每天自动汇总当天条目，生成一条「日报 / 日记」式记录：
   - 日期
   - 一个笔记式标题（概括这一天）
   - 今天做了哪些事
   - 哪些重要、哪些不重要
   - 提炼出的思考 / 待办

   触发方式：当天结束时（定时）或打开记事本时按需生成；异步、用轻量模型；可手动编辑。

### 差异点（对标已有产品）

| 产品 | 形态 |
|---|---|
| [AudioPen](https://www.audiopen.ai/) | 专门去录 → 润色成笔记 |
| [Voicenotes](https://nubiapage.com/voicenotes-review-2026-ai-app-pricing-login-user-experience-and-faqs/) | 语音知识库 + 可搜索 + 接 Obsidian/Notion |
| AudioNotes / TalkNotes | 转写 + 分类 + 模板归档 |

它们都是**「专门去记笔记」的独立 App**。PunkType 的不同：**记录是日常输入的副产品**，被动收录、完全本地私密——这个角度上面竞品都没有。

### 架构契合（不是新 App，是现有东西的进化）

- 已有 `HistoryManager`，**每次出字都已存储**。
- 记事本 = HistoryManager + ① 异步 AI 分类打标签 + ② 独立窗口（快捷键唤起）可看 / 编辑 / 搜索 / 导出。

### 已拍板（v1 规格）

- **范围**：地基 + 每日总结一起做。
- **收录**：被动收录**所有听写**（极速 / 润色 / 格式三档的出字都收，每条保留）；**「选中文字命令」的结果不收**（那是处理别人的文字，不是用户表达）。
- **打开**：专属快捷键 + 菜单栏入口。
- **每日总结**：自动生成「日报」（日期 + 笔记式标题 + 今天做了什么 + 重要性 + 思考/待办）。
- **存储**：按天分文件（`notebook/YYYY-MM-DD.json`），去掉 100 条上限，保留全部。

### v1 待定（小，做的时候顺带定）

- 日报触发：打开当天 / 手动按钮（v1 先不做定时后台）。
- 标签 / 自动分类（放 v1.2）。
- 隐私开关（某些 App 不收录）、导出到 Markdown/Obsidian（远期）。

### v1 施工图

1. `NotebookStore`：按天加载 / 追加条目；条目新增 `sourceApp`、可编辑 `text`、可选 `summary`(按天)。
2. 出字处接入：在 polish/format/fast 出字后写入当天文件（排除命令模式）。
3. `NotebookWindow`：左侧按天列表、右侧当天流水 + 日报、顶部搜索。
4. 每日总结：当天条目 → 轻量模型 → 结构化日报，异步、可手动重生成。
5. 快捷键 + 菜单栏入口；设置里原 History 页改为「打开记事本」。

## 六、任务清单与优先级（Backlog）

> 状态：✅ 已发布 / 🔨 进行中 / ⬜ 待办

### P0 — 先收尾（bug 优先于新功能）

- 🔨 **润色偶发卡顿排查**：已加诊断日志（opt-in）+ 流式空闲超时 30s→20s。等抓到一次卡顿日志定位真因后修复，再随版发布。当前为本地 WIP，未提交。

### 已完成（v1.3.x）

- ✅ **记事本**（被动收录 + 每日总结 + 启动补生成 + 排除名单 + 窗口），见 v1.3.0。
- ✅ **App 感知**（仅润色档）、**风格画像**、**Fn/⌥Space 快捷键**、**权限引导窗**。

### P1 — 开放骨架（模式包 + 商城）

- ⬜ **配置包导出 / 导入**：把一档（或全部）导出成 `.json`，含命名、提示词、模型、引擎、流式、词典、App 规则；可导入成一个可选档位。官方先放示范包（正经 / 深思 / 诡辩…）。
- ⬜ **GitHub 轻量商城 + 审核**：`packs/` + `index.json`；社区在 GitHub 提 PR 贡献包，作者审核合并后，App 内拉清单一键导入/切换。零后端。

### P2 — 快捷键 = 动作层（见第七节）

- ⬜ **一键切档**：分别给 ⚡极速 / ✨润色 / 📄格式 各绑一个快捷键，按一下直接切到该档并录音。
- ⬜ **动作快捷键**：专用「翻译」键、专用「询问」键；后续可配置更多「热键 → 动作」。
- ⬜ **热键→动作映射框架**：把"哪个键触发什么"做成可配置表，模式包也能声明自带快捷键。

### P3 — 体验补强（低成本，日常实用）

- ⬜ **撤销上次插入**：一个热键撤销刚才那次出字。
- ⬜ **词典替换 / 缩写展开**：说「我邮箱 / 公司地址 / 标准签名」→ 自动展开；品牌词强制写法。

### P4 — 按需 / 远期

- ⬜ **音频文件转写**、**多语言 / 自动语种检测**、**后处理 Hook 开放第三方**、**企业证书签名（治本免重授权）**。

### 推荐顺序

先 **P1 配置包导出/导入**（开放地基，最小可行）→ **P2 一键切档 + 翻译/询问键**（日常最顺手，且为"热键=动作"打底）→ **商城** → **P3 小功能**穿插。

## 七、快捷键 = 动作层（hotkey → action）

PunkType 从「一个键 → 在当前档位录音」演进为「**多个键，每个键绑一个动作**」。统一抽象：

```
快捷键  →  动作
          ├─ 切到某档并录音：⚡极速 / ✨润色 / 📄格式 / 任意自定义模式包
          ├─ 翻译：口述 → 翻成目标语言 → 插入
          ├─ 询问：口述一个问题 → 问 AI → 结果（插入或弹窗）
          └─ 以后更多：撤销、打开记事本、某个专用模式包…
```

- **模式（含模式包）本身就是一种动作**，所以「一键切到润色」和「一键翻译」是同一套机制的两种绑定。
- **模式包可以自带快捷键声明**（导入即带快捷键），和商城打通。
- 维护一张可配置的「热键 → 动作」表，用户/包都能往里加。

### 待澄清（做之前定）

- **翻译键**：是"口述 → 翻译 → 插入"，还是"翻译当前选中文字"？（命令模式已能翻译选中，所以翻译键更可能是前者：直接口述出译文）
- **询问键**：问 AS 后结果**插入光标**还是**弹窗展示**？（弹窗更像助手，插入更像写作）
- **目标语言**：翻译键固定一个目标语言，还是循环切换/记忆上次？
